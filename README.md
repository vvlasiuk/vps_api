# VPS API Confirmation Server

## Опис
FastAPI сервер для підтвердження команд через HTML‑кнопки, з інтеграцією RabbitMQ та MariaDB, дворівневою системою токенів і логуванням помилок.

### Основні можливості
- Прийом команд через POST /command (тимчасові токени)
- Відправка команд у RabbitMQ (input.queue)
- Видача тимчасових токенів через POST /token (по master-токену)
- Управління контекстом через /context
- Логування помилок у файл та sys_error.queue
- Адміністрування master-токенів через CLI

### Старт
1. Швидкий старт (Windows):
   Запустіть скрипт
   ```bat
   setup.bat
   ```
   Він створить віртуальне середовище (venv) та встановить залежності автоматично.

2. Встановіть залежності вручну (альтернатива):
   ```bash
   pip install -r requirements.txt
   ```
3. Налаштуйте змінні середовища для MariaDB та RabbitMQ 
4. Проведіть міграції (alembic або вручну):
   Увага!
   Після налаштування змінних середовища і перед запуском сервера обов’язково виконайте команди з розділу міграцій (Alembic) цього README, щоб створити всі необхідні таблиці у базі даних.

    - Якщо використовується Alembic, виконайте команду:
       ```bash
       alembic upgrade head
       ```
       Це створить необхідні таблиці у вашій базі даних згідно з моделями.
    - Якщо alembic не використовується, створіть таблиці вручну відповідно до моделей у app/models.py (SQL-структура описана у класах Base, MasterToken, Token, Context).
5. Запустіть сервер:
   ```bash
   uvicorn app.main:app --host 0.0.0.0 --port 8000 --ssl-keyfile=key.pem --ssl-certfile=cert.pem
   ```
6. Для керування master-токенами використовуйте CLI:
   ```bash
   python manage_tokens.py --help
   ```

### Структура RabbitMQ
- **input.queue**: приймає команди
- **sys_error.queue**: приймає помилки

### Безпека
- Тільки HTTPS (самопідписані сертифікати)
- Авторизація через токени

Опис таблиць бази даних
**master_tokens**  
Зберігає master-токени для виконання запитів з скриптів та видачі тимчасових токенів.
- id: autoincrement ключ
- token: унікальний токен
- description: опис
- status: статус (active/revoked)
- created_at: дата створення

**tokens**  
Тимчасові токени для виконання команд з зовнішніх html файлів.
- id: autoincrement ключ
- token: унікальний токен
- command_name: назва команди
- command_params: параметри команди
- created_at: дата створення
- expires_at: дата закінчення дії
- issued_by: id master-токена
- usage_count: кількість використань
- max_uses: максимальна кількість використань
- last_used_at: останнє використання
- context_id: ідентифікатор контексту

**context**  
Контекст виконання команд (наприклад, сесія користувача чи бізнес-процес).
- id: autoincrement ключ
- object_id: унікальний ідентифікатор обʼєкта
- context_data: довільні дані (JSON)
- created_at: дата створення
- updated_at: дата оновлення
- end_at: дата завершення
- closed: ознака закриття

**users**  
Користувачі системи.
- id: autoincrement ключ
- lastname, firstname, middlename: ПІБ
- position, department, city: додаткові поля
- phone, email, chat_id: контакти
- role, username: роль та логін
- created_at: дата створення

**global_message_context**  
Звʼязок між глобальним повідомленням та контекстом. Глобальні повідомлення це наприклад єдиний 
звязок між розсилкою, коли одне те ж повідомлення або кнопку інлайн відправляємо декільком користувачам. Тобто це одне повідомлення а відправлене декільком щоб потім мати можливість отримати наприкла ід повідомлень і змінити їх разом
- global_msg_id: autoincrement ключ
- context_id: ідентифікатор контекст

**global_message_telegram**
повідомлення відправлені в телеграм якщо в межах глобального повідомлення всим. 
можнга отримувати всі повідомлення що були відправлені в межах одного сповіщення різним особам.
